- **Token port.** All `claude-tokens.css` `:root` tokens + `.dark` overrides ported into `frontend/app/globals.css`, plus the `:root` brand layer from `tokens.css` (compatibility aliases + the `--river-*` / `--river-ink` additions). - **Tailwind v4 theme split.** Fonts live in a regular `@theme {}` (emitted to `:root`, so they serve both raw `var(--font-*)` in later component CSS and the `font-sans/serif/mono` utilities); the palette lives in `@theme inline {}` so `bg-*`/`text-*` utilities reference the live `var()` and the `.dark` swaps flow through. Keeping font stacks out of the inline block avoids a self-referential `--font-sans: var(--font-sans)` cycle.
- **`@layer base`.** Base element + `.t-*` type rules are wrapped in `@layer base` so Tailwind utilities always win. In Tailwind v4 cascade-layer order outranks specificity, so an unlayered bare `h1 {}` would otherwise beat a `text-3xl` utility and silently break later slices.
- **Orange-leak guard.** Used the hybrid's `a { color: inherit }`, NOT `claude-tokens`' `a { color: var(--accent) }` (which is Crail orange). Porting the Claude base verbatim would have leaked orange onto every link, breaking the "orange only on the Powered-by-Claude badge" rule (¬ß9 audit).

### Slice 4 (formatters) ‚Äî decisions + discoveries

- **Anchored to the prototype, not the plan's guesses.** The design source of truth (`design/.../prototype-hybrid/components.jsx`) defines the inline helpers, so the formatters match them exactly rather than the speculative examples in the old "Next slice" note. Resolved open questions: currency is **USD `$`** (backend field is `cost_usd`), **3 decimals** (`fmtCost = $${usd.toFixed(3)}` ‚Üí `$0.078`, not the note's 2-dp `$0.08`); and `42180` ms ‚Üí **`42.2s`** (not `42s`).
- **`formatCurrency(usd)`** = `` `$${usd.toFixed(3)}` `` ‚Äî always 3 dp so the metrics column stays aligned (`$0.000`, `$1.500`).
- **`formatDuration(ms)`** mirrors `fmtMs`'s three tiers: `<1s` ‚Üí `"<n>ms"`; `<1m` ‚Üí `"<n.n>s"` (one decimal); else ‚Üí `"<m>m <s>s"` (whole seconds, `Math.round`). Boundaries pinned in tests: `999`‚Üí`999ms`, `1000`‚Üí`1.0s`, `60000`‚Üí`1m 0s`, `125000`‚Üí`2m 5s`.
- **`formatConfidence(score)`** = `` `${Math.round(score * 100)}%` ``. The prototype has no single confidence helper ‚Äî the compact bar shows `pct = Math.round(score*100)` (bare number) and the expanded bar/stats show `score.toFixed(2)`. Slice 4's brief defines this formatter as `0.86 ‚Üí 86%`, so it returns the bar's own `pct` **plus the unit**: the canonical labelled percentage for mini-stats/labels. Components that need the bare number or 2-dp decimal still compute those locally.
- **Scope decision ‚Äî no `null` handling.** Kept `formatConfidence` as `number ‚Üí string`. A failed answer (`confidence: null`) renders **no score bar at all** in the design, and the adapter already carries `failed`/`needs_review`/`review_reason`, so the component guards null ‚Äî the formatter stays focused on numeric scores. (Avoids inventing a placeholder glyph and the banned em dash.)

### Slice 5 (upload validation) ‚Äî decisions + discoveries

- **Anchored to the prototype's `Dropzone.accept`** (`design/.../prototype-hybrid/components.jsx:177-183`), not the plan's guesses. Pinned exactly: check **TYPE first, then SIZE**; ceiling is `10 * 1024 * 1024` (10 MiB, binary ‚Äî `fmtBytes` divides by 1024¬≤ and labels "MB") with a strict `>` so **exactly 10 MiB passes** and one byte over fails; messages are verbatim ‚Äî type ‚Üí `"We couldn't read this file. - **API shape (a design call ‚Äî ¬ß7 only said "with messages"):** `validateUpload(file)` returns a discriminated union `{ ok: true } | { ok: false; reason: "type" | "size"; message: string }` rather than throwing ‚Äî a rejected upload is an expected outcome the UI renders, so the component branches on `ok` with full type-narrowing and no try/catch. `reason` lets the Dropzone style/label by failure mode without re-parsing the message.
- **Param type `UploadFile = Pick<File, "name" | "size">`.** A real browser `File` satisfies it, but tests pass a light `{ name, size }` stand-in so the 10 MB boundary case allocates nothing. The validator only ever needs those two fields.

Extension match is `ACCEPTED_EXTENSIONS.some(ext => name.toLowerCase().endsWith(ext))` ‚Äî equivalent to the prototype's `/\.(pdf|xlsx)$/i` for these extensions (case-insensitive, end-anchored: `report.PDF` passes, `malware.pdf.exe` is rejected) but derived from the constant list so the accept attribute and the check can't diverge.

### Slice 17 tweak #6 (visible light/dark toggle + finish the dark theme) ‚Äî decisions + discoveries

- **The dark theme was 90% done in tokens; this slice added the control + the last 10%.** Slice 2 already ported the `:root.dark, .dark` token overrides into `globals.css`, so flipping a `.dark` class on `<html>` swaps every component that reads `var(--bg-*)`/`var(--fg-*)`/`var(--border-*)` ‚Äî which is almost all of them. The work was a visible control + a no-flash boot + the handful of surfaces that hardcode light paper/white.
- **No `@custom-variant dark` needed ‚Äî nothing uses `dark:`.** Grepped the tree: **zero** Tailwind `dark:` utilities anywhere. The theme is driven entirely by the CSS-variable swap behind the `.dark` selector, so wiring the Tailwind `dark:` variant to the class (the Slice 2 deferred note) would be a speculative abstraction with no consumer. Left it out per the "no speculative abstractions" rule; revisit only if a `dark:` utility is ever introduced.
- **`useSyncExternalStore`, not `useState` + `useEffect`.** The first cut read the `.dark` class in a mount effect (`setIsDark(...)`), but ESLint's `react-hooks/set-state-in-effect` flags that as an **error** (CI/pre-commit would block). The `.dark` class is genuinely external browser state, so the React-blessed fix is `useSyncExternalStore(subscribe, getSnapshot, getServerSnapshot)`: `getServerSnapshot ‚Üí false` so the SSR HTML and first client paint agree (no hydration mismatch), `getSnapshot` reads the live class, and a `subscribe` listening for a same-tab `isq-theme-change` event + cross-tab `storage` keeps it in sync. Cleaner than the effect, lint-clean, and free cross-tab sync.
